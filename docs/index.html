MarginNote 4 URL API 使用文档

欢迎使用 Url Apis 插件！本插件为 MarginNote 4 提供了强大的 URL Scheme 接口，允许您通过快捷指令、脚本或其他应用（如 Obsidian）直接读取、修改和管理您的 MarginNote 笔记。

1. 基础概念

1.1 密钥与权限组 (Secret & Permissions)

为了保护您的数据安全，所有 API 请求都需要进行身份验证。在 MarginNote 中点击插件图标可打开 Secret 管理面板，您可以在面板中完成以下操作：

  • 生成密钥：点击权限按钮（EMPTY / READ / WRITE / ADMIN / ALL）即可生成对应权限的新密钥，并自动复制到剪贴板。
  • 查看密钥列表：左侧区域列出所有已生成的密钥，点击可选中查看详情。
  • 查看详情：右侧显示选中的密钥 ID、密钥（部分掩码）、权限组（Permission）、备注（Note）。
  • 添加备注：在 Note 输入框中填写备注文字，点击 SAVE NOTE 保存。
  • 复制密钥：点击 COPY 将选中密钥复制到剪贴板。
  • 删除密钥：点击 DELETE 删除选中的密钥。
  • 修改权限：在 SET PERMISSION 区域点击对应权限按钮，更改选中密钥的权限组。

密钥支持的权限组：

  • 空权限 (EMPTY)：无法执行任何动作。
  • 只读权限 (READ)：仅允许读取（ping, read, ls, find, tree）。
  • 写入权限 (WRITE)：支持读取和修改（上述权限 + write）。
  • 管理权限 (ADMIN)：支持所有操作，包含危险操作（上述权限 + delete）。
  • 全部权限 (ALL)：通配全部动作。

1.2 路径模型 (Path Resolver)

API 使用一种直观的 URI 路径模型来定位笔记本和卡片：

  • notebook://：根路径（所有笔记本）。
  • notebook://笔记本名称：定位到特定笔记本。
  • notebook://笔记本名称/父卡片名/子卡片名：按标题层级定位卡片。
  • @id:noteId：直接通过卡片的内部 ID 定位卡片。
  • @current：动态定位到您当前正在学习的窗口中聚焦的笔记或笔记本。
  • @root：等同于 notebook://。

1.3 防重放机制 (Replay Guard)

为防止恶意应用截获 URL 后重复执行相同请求，系统要求每次请求必须携带独一无二的 requestId。同一密钥下执行过的 requestId 在短时间（通常为2分钟）内会被拒绝重复执行。

2. URL 的构建

所有的 API 请求都通过标准的 URL Scheme 触发，基础入口为：

marginnote4app://addon/api

您需要通过 URL Query (GET 参数) 的形式传递控制字段：

参数名 是否必须 说明
requestId 必须 请求唯一标识符，例如 UUID 或带毫秒的时间戳，用于防重放。
action 必须 要执行的操作名称，如 read、write 等。
secret 必须 您的专属鉴权密钥，例如：mnsec_xxx...
x-success 可选 成功时的回调 URL。缺少时请求仍会执行，但不会返回成功结果。
x-error 可选 失败时的回调 URL。缺少时请求仍会执行，但失败结果只会记录日志。
payload 按需 操作的具体参数，必须为 JSON 字符串并进行 URL Encode。

构建示例：

如果您想触发一次 ping 请求，您的 URL 将如下所示：

marginnote4app://addon/api?requestId=req_20260529163045123&action=ping&secret=mnsec_abc&x-success=myapp://success&x-error=myapp://error

3. 回调与响应格式

请求处理完成后，如果提供了 x-success 或 x-error，插件会打开回调应用，并将结果转为 JSON 追加在 payload 参数中：

myapp://success?payload=%7B%22code%22%3A%22OK%22...%7D

解开 URL Encode 后的响应 JSON 结构：

{
  "code": "OK",
  "message": "OK",
  "requestId": "req_20260529163045123", // 与您请求时一致
  "data": { ... }, // 成功时包含的返回数据
  "details": null // 错误时的详细原因
}

完整的错误码：

错误码 说明
OK 成功。
BAD_REQUEST 请求参数错误（如缺少必填字段）。
AUTH_FAILED 密钥认证失败（secret 无效）。
FORBIDDEN 权限不足（密钥不具备该操作所需的权限）。
NOT_FOUND 目标路径或资源不存在。
REPLAY_DETECTED 防重放机制触发，该 requestId 已被使用过。
INTERNAL_ERROR 插件内部异常，请检查日志。

4. Action (可用动作详解)

ping

说明：健康检查，测试密钥和网关是否可用。无需 Payload。

返回：版本号 version、服务器时间 time 和 ok: true。

ls (列表查询)

说明：列出指定路径下的笔记本或笔记项。

{
  "path": "notebook://测试笔记本", // 默认 notebook://
  "depth": 1, // 展开层级，默认 1
  "type": "all" // 返回类型: all | notebook | note
}

read (读取详情)

说明：读取目标笔记本或某张卡片的详细信息（包含标题、正文、评论及子节点信息）。

{
  "path": "@current" // 必须提供，支持 @id:xxx 或 notebook://xxx
}

find (全局搜索)

说明：根据关键词在指定范围内搜索卡片或笔记本，匹配标题或内容。

{
  "pattern": "搜索关键词", // 必须提供
  "scope": "notebook://", // 搜索范围，默认为全局
  "type": "note" // 搜索目标: all | notebook | note
}

tree (获取树状结构)

说明：导出层级结构的笔记树和带缩进的文本渲染。

{
  "path": "notebook://测试笔记本", 
  "depth": 3 // 获取的深度，默认 3
}

write (修改与写入)

说明：修改笔记数据。根据 mode 参数区分为创建、更新、追加和移动操作。

创建卡片 (create):

{
  "mode": "create",
  "path": "notebook://笔记本名称", // 父节点路径（也可以用 parentPath 字段）
  "title": "新卡片标题", // 必须
  "content": "摘录或正文内容",
  "colorIndex": 0, // 可选：修改颜色
  "fillIndex": 0   // 可选：修改填充样式
}

更新卡片 (update):

{
  "mode": "update",
  "path": "@id:noteId", // 需要修改的笔记目标
  "title": "新标题", 
  "content": "新正文",
  "colorIndex": 0, // 可选：修改颜色
  "fillIndex": 0   // 可选：修改填充样式
}

追加评论 (append):

{
  "mode": "append",
  "path": "@id:noteId",
  "content": "追加的评论文本", // 必须
  "markdown": true // 可选：是否作为 markdown 格式追加
}

移动卡片 (move):

{
  "mode": "move",
  "path": "@id:noteId", // 要移动的卡片
  "dstPath": "notebook://笔记本名/新的父卡片名" // 目标路径
}

delete (删除笔记)

说明：删除指定卡片（需具备 ADMIN 权限组的密钥）。

{
  "path": "@id:noteId", // 必须提供，被删除卡片目标
  "recursive": false, // 可选：是否连同子节点一起删除
  "force": false // 可选：强制删除
}

注意：默认情况下，如果卡片包含子节点，删除将被拒绝，除非提供 recursive: true 或 force: true。
